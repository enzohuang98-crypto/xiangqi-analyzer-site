Security baseline

C 級資安說明

版本 1.0｜更新日期：2026 年 6 月 15 日

「C 級」是本專案自行定義的靜態產品網站安全基線，不是政府分級，也不代表 ISO 27001、SOC 2、滲透測試或任何第三方認證。

適用範圍

本基線只涵蓋這個公開說明網站。桌面 App、未來的安裝檔、更新服務、第三方象棋引擎及 AI Provider 有各自的風險與安全控制，不因網站符合本基線而自動符合。

網站架構

  • 純靜態 HTML 與 CSS，不含後端、資料庫或伺服器端程式。
  • 不執行 JavaScript，不使用第三方套件、CDN、外部字型或遠端圖片。
  • 不提供登入、註冊、留言、搜尋、付款、檔案上傳或資料輸入表單。
  • 目前不放置安裝檔，也不提供自動下載。

瀏覽器端限制

每一頁都設定嚴格的 Content Security Policy。預設拒絕所有資源，再僅允許同來源 CSS、圖片與字型；腳本、網路連線、框架、外掛物件、背景工作、媒體與表單提交均被禁止。

網站同時使用 no-referrer，避免使用者從本站前往其他網站時傳送目前頁面網址。 外部連結若日後加入，必須使用 noopener 與 noreferrer。

資料與隱私

  • 本站不建立 Cookie、Local Storage 或 IndexedDB 資料。
  • 本站不嵌入分析工具、廣告網路、社群像素或客服追蹤程式。
  • 本站不要求、接收或處理 API Key、密碼、棋譜或個人資料。
  • 代管平台可能基於維運與安全目的記錄連線資訊，受其政策約束。

供應鏈與發布

  • 網站不使用 npm 套件與自動下載的前端依賴。
  • 原始檔以 Git 版本控制，變更須通過本機網站檢查後才發布。
  • 網站 Repository 與桌面 App Repository 分離，避免公開主程式原始碼或機密設定。
  • 任何憑證、API Key、Token、私鑰與安裝簽章材料均不得提交到網站 Repository。

發行安裝檔前的必要條件

未來若要提供下載，至少需要完成以下項目，才可維持 C 級發行基線：

  • 安裝檔使用可信任的程式碼簽章憑證簽署。
  • 每個版本公開 SHA-256 雜湊與明確版本號。
  • 下載連結只允許 HTTPS，且指向受控的正式發布位置。
  • 發布前進行惡意程式掃描、依賴漏洞檢查與乾淨環境安裝測試。
  • 舊版本不得被靜默替換；任何檔案變更都必須產生新版本與新雜湊。

已知限制

GitHub Pages 不允許專案自行設定所有 HTTP 回應標頭。因此 X-Content-Type-Options、完整的 Permissions-Policy 與 CSP 的 frame-ancestors 等控制，需在未來使用可自訂回應標頭的代管服務時補強。 現階段以無腳本、無輸入、無敏感資料及頁面內 CSP 降低風險。
